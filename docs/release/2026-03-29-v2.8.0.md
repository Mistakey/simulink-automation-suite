# v2.8.0 — Physical Connection Visibility

## Summary

- `connections` action now traverses SPS physical ports (LConn/RConn) in addition to Simulink signal lines
- Physical neighbors reported in a dedicated `physical_neighbors` field; signal edges gain a `type` field (`"signal"` / `"physical"`)
- Backlog cleared: B-001 resolved

## Highlights

### Physical connections now visible via `connections`

Previously `connections` only tracked Simulink signal lines; SPS electrical topology was invisible. With v2.8.0:

- **`physical_neighbors`** — new summary field listing all physical-network neighbors (present only when physical connections exist)
- **`type` field on edges** — every edge in `detail: "ports"` / `detail: "lines"` output now carries `"type": "signal"` or `"type": "physical"`
- Physical connections are **direction-agnostic**: the `direction` parameter (`upstream` / `downstream` / `both`) applies only to signal edges; physical neighbors are always reported regardless of direction

Example response for a PSrc block connected to PLoad1 and PLoad2 via SPS physical lines:

```json
{
  "target": "my_model/PSrc",
  "direction": "both",
  "depth": 1,
  "detail": "summary",
  "upstream_blocks": [],
  "downstream_blocks": [],
  "physical_neighbors": ["my_model/PLoad1", "my_model/PLoad2"]
}
```

With `detail: "ports"`:

```json
{
  "edges": [
    {"src_block": "my_model/PSrc", "src_port": 1, "dst_block": "my_model/PLoad1", "dst_port": 1, "signal_name": "", "type": "physical"},
    {"src_block": "my_model/PSrc", "src_port": 1, "dst_block": "my_model/PLoad2", "dst_port": 1, "signal_name": "", "type": "physical"}
  ],
  "physical_neighbors": ["my_model/PLoad1", "my_model/PLoad2"]
}
```

## Compatibility / Upgrade Notes

- **Additive only** — `upstream_blocks`, `downstream_blocks`, existing edge fields unchanged
- `physical_neighbors` is omitted from the response when no physical connections are found (no action required for models without SPS)
- `type` field is new on every edge; existing consumers that do not read `type` are unaffected

## 中文说明

### 本次更新

SPS 电气连接（LConn/RConn 物理端口）现在可通过 `connections` action 查看，这是 B-001 缺陷的修复。

### 主要变化

- **`physical_neighbors` 新字段**：summary 输出中新增物理连接邻居列表，仅在存在物理连接时出现
- **边的 `type` 字段**：`detail: "ports"` 或 `detail: "lines"` 时，每条边携带 `"type": "signal"` 或 `"type": "physical"`
- 物理连接无方向之分，`direction` 参数不影响物理邻居的输出

### 兼容性

纯增量改动，旧有字段不变，无需修改现有调用。无 SPS 连接的模型响应结构与之前完全相同。
