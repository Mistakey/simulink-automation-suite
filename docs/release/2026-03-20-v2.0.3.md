# Release v2.0.3

Release date: 2026-03-20

## Summary

`v2.0.3` is a patch release that strengthens the agent-safe single-parameter edit loop for `set_param`.

## Highlights

- Dry-run preview now returns a machine-executable `apply_payload` in addition to `rollback`.
- Execute mode now supports guarded replay through `expected_current_value`, preventing stale previews from mutating the model.
- Added explicit top-level error codes for guarded edit recovery:
  - `precondition_failed`
  - `verification_failed`
- Added structured recovery metadata on write failures, including `write_state`, `safe_to_retry`, and `recommended_recovery`.
- Updated `simulink-edit` docs and README examples to describe the canonical loop:
  - `inspect`
  - `set_param` dry-run
  - replay `apply_payload`
  - `inspect`
  - replay `rollback` if needed

## Compatibility / Upgrade Notes

- Plugin version is now `2.0.3`.
- Schema version remains `2.0` because the plugin major.minor contract did not change.
- No new action was added; this release strengthens `set_param` runtime semantics and recovery signaling.
- Callers that already consume `set_param` should account for:
  - preview responses including `apply_payload`
  - stale execute replays returning `precondition_failed`
  - verification mismatch returning `verification_failed`

## 中文说明

### 摘要

`v2.0.3` 补丁版本，强化 `set_param` 的代理安全单参数编辑循环。

### 重点变更

- 预览（dry-run）现返回可机器执行的 `apply_payload`（除原有 `rollback` 外）。
- 执行模式支持通过 `expected_current_value` 进行守护重放，防止过期预览意外修改模型。
- 新增顶层错误码：`precondition_failed`（前置条件失败）、`verification_failed`（验证失败）。
- 新增写入失败的结构化恢复元数据：`write_state`、`safe_to_retry`、`recommended_recovery`。
- 文档更新为规范的 inspect → dry-run → replay apply_payload → inspect → rollback 循环。

### 升级提示

- 插件版本升至 `2.0.3`，Schema 版本仍为 `2.0`。
- 已有 `set_param` 调用方需关注：预览响应新增 `apply_payload`；过期重放返回 `precondition_failed`；验证不一致返回 `verification_failed`。
