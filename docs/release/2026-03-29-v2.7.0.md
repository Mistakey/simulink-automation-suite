# v2.7.0 Release Notes

## Summary

Three backlog items resolved: physical port support for SPS electrical connections (F-003, P0), atomic multi-parameter set_param (F-006, P1), and model compilation diagnostics capture (I-003, P1). All changes are additive with no breaking changes.

## Highlights

- **Physical port names (F-003)**: `line_add` and `line_delete` now accept string port identifiers (e.g. `"RConn1"`, `"LConn1"`) in addition to integers. This unblocks all SimPowerSystems electrical connections via CLI. New `"port"` JSON type accepts both `int` and `string`.
- **Atomic multi-param set_param (F-006)**: New `params` object field enables setting multiple parameters in a single MATLAB call, avoiding intermediate-state validation errors (e.g. `rep_seq_t` and `rep_seq_y` must match in length). Full guarded write safety: dry-run preview with `changes` array, precondition guard via `expected_current_values`, per-param read-back verification, rollback captures all original values.
- **Compilation diagnostics (I-003)**: `model_update` now captures MATLAB command window output during `set_param(..., 'SimulationCommand', 'update')` via `evalc`. Response includes a `diagnostics` array with parsed output lines, giving AI agents visibility into compilation warnings beyond just the last `lastwarn`.

## Compatibility / Upgrade Notes

- All changes are additive. No breaking changes to existing action schemas or response shapes.
- `line_add` / `line_delete`: `src_port` and `dst_port` field type changed from `"integer"` to `"port"`. Existing integer values continue to work; string values are now additionally accepted.
- `set_param`: `param` and `value` fields changed from required to conditionally required (must provide either `param`+`value` or `params`, not both). Existing single-param payloads work unchanged.
- `model_update`: Response now includes `diagnostics` array (empty list on clean update). The existing `warnings` field is unchanged.
- New fields: `params`, `expected_current_values` (set_param); `diagnostics` (model_update).
- New JSON types: `"port"` (int or string), `"object"` (dict).
- Schema version bumped from `2.6` to `2.7`.

## Validation

- 411 unit tests passing (44 new across all three features)
- Schema introspection verified: new fields and types registered correctly
- Docs contract verified: reference.md updated with multi-param and diagnostics response shapes
- Manifest validation: plugin.json and marketplace.json version-synced
- No live test report for this release (live test report is from v2.5.0; transport-layer `evalc` changes require live MATLAB verification)

## 中文说明

### 概要

解决三个 backlog 条目：SPS 物理端口支持（F-003）、原子多参数 set_param（F-006）、模型编译诊断输出（I-003）。

### 亮点

- **物理端口名称（F-003）**：`line_add` 和 `line_delete` 现在接受字符串端口名（如 `"RConn1"`、`"LConn1"`），解锁了所有 SPS 电气连接的 CLI 操作。
- **原子多参数更新（F-006）**：新增 `params` 对象字段，支持一次调用设置多个参数，避免中间态校验报错（如 Repeating Sequence 的时间/输出向量长度必须匹配）。完整的安全写入流程：预览→前置条件检查→原子写入→逐参数回读验证→回滚。
- **编译诊断（I-003）**：`model_update` 通过 `evalc` 捕获 MATLAB 编译输出，返回 `diagnostics` 数组，AI 可据此定位具体的编译警告。

### 兼容性

所有变更均为增量式，无破坏性变更。现有调用方式完全兼容。
