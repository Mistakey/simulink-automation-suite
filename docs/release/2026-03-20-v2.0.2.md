# Release v2.0.2

Release date: 2026-03-20

## Summary

`v2.0.2` is a patch release that hardens GitHub auto release publishing and cleans up AI-agent guidance boundaries for release work.

## Highlights

- Added a GitHub Actions release workflow that publishes on `vX.Y.Z` tags and supports `workflow_dispatch` for controlled backfill.
- Added deterministic release validation and release notes scripts:
  - `scripts/check_release_metadata.py`
  - `scripts/build_release_notes.py`
- Clarified release-notes sourcing so curated `docs/release/*vX.Y.Z*.md` files win, with deterministic git-history fallback when no doc exists.
- Moved detailed release policy out of `.claude/CLAUDE.md` and into `.claude/rules/release.md`, so AI agents use the release rule as the single detailed source of truth.
- Updated Codex repository guidance to explicitly allow scoped subagent delegation for review and parallel work.

## Compatibility / Upgrade Notes

- Plugin version is now `2.0.2`.
- Schema version remains `2.0` because the plugin major.minor contract did not change.
- No CLI contract or skill-surface migration is required for existing installs.
- `v2.0.1` remains the prior release tag; `v2.0.2` is the correct tag for the auto-release and agent-guidance fixes.

## 中文说明

### 摘要

`v2.0.2` 补丁版本，强化 GitHub 自动发布流程并明确 AI 代理在发布工作中的指导边界。

### 重点变更

- 新增 GitHub Actions 发布工作流，支持 `vX.Y.Z` 标签触发发布和 `workflow_dispatch` 手动回补。
- 新增确定性发布验证和发布说明构建脚本：`check_release_metadata.py`、`build_release_notes.py`。
- 明确发布说明来源优先级：人工撰写的 `docs/release/*vX.Y.Z*.md` 优先，无文档时回退到 git 历史自动生成。
- 将详细发布策略从 `.claude/CLAUDE.md` 迁移至 `.claude/rules/release.md`，作为 AI 代理的唯一发布规则来源。

### 升级提示

- 插件版本升至 `2.0.2`，Schema 版本仍为 `2.0`。
- 无需 CLI 契约或技能接口迁移。
