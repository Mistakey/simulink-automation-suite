# Release v2.9.0

**Date**: 2026-03-29
**Schema**: 2.9 (22 actions, 35 error codes)

## Summary

Comprehensive backlog sweep addressing all issues discovered during the FOC model building exercise. Adds batch operations for block_add/line_add, a new model_copy action, simulation workspace storage, a dedicated post-simulation analysis agent, and numerous CLI/documentation improvements.

## Highlights

### New Action: `model_copy`
Copy a loaded model to a new file path without renaming the original. Uses file-level `copyfile` to avoid MATLAB's `save_system` rename side-effect.

### Batch Mode for `block_add` and `line_add`
Both actions now accept an array field (`blocks` / `lines`) for batch operations. Mutually exclusive with single-item fields. Processes items sequentially with stop-on-first-failure semantics. Reduces token consumption significantly when building models (20 blocks = 1 call instead of 20).

### Simulation Workspace Storage (FEAT-002b)
`simulate` now stores the `SimulationOutput` object in the MATLAB base workspace as `sl_sim_result`. This enables post-simulation analysis via `matlab_eval` or the new `sim-analyst` agent.

### New Agent: `sim-analyst`
Dedicated subagent for post-simulation data analysis. Reads `sl_sim_result` from workspace, writes and executes analysis code (MATLAB via `matlab_eval` or Python scripts), returns structured conclusions. Isolates potentially massive waveform data from the main conversation context.

### `simulate` Timeout Parameter
New `timeout` field for long-running simulations (SPS discrete models). Returns `simulation_timeout` error code on expiry.

### `block_add` Cross-Model Source
`source` field now accepts loaded model paths (e.g. `RefModel/Controller`) in addition to library paths. Enables reusing validated subsystems across models.

### `source_not_found` Path Suggestions
When `block_add` fails with `source_not_found`, the error response now includes a `suggestions` field with similar library paths found via `difflib` fuzzy matching.

### CLI: `--json-file` Option
New `--json-file <path>` argument reads JSON payload from a file, bypassing Windows shell quoting issues with `--json`.

### UTF-8 Output
`emit_json` now uses `ensure_ascii=False`, properly rendering CJK characters instead of `\uXXXX` escapes.

## Compatibility / Upgrade Notes

- **Schema version**: 2.7 -> 2.9
- **New error codes**: `model_copy_failed`, `simulation_timeout`
- **Field changes**: `block_add` `source`/`destination` changed from required to optional (required-ness now depends on whether `blocks` batch field is used). Same for `line_add` `src_block`/`src_port`/`dst_block`/`dst_port`.
- **No breaking changes**: All existing single-mode calls work identically. Batch mode is additive.

## 中文说明

### 概要

全面清理 FOC 模型搭建实践中发现的所有问题。新增批量操作、模型复制、仿真数据存储、仿真分析 agent，以及多项 CLI 和文档改进。

### 主要更新

- **`model_copy`** — 新 action，文件级复制，不重命名原模型
- **批量模式** — `block_add` / `line_add` 支持数组字段，一次调用添加多个块/线
- **仿真结果存储** — `simulate` 后结果存入 workspace `sl_sim_result`，支持后续分析
- **`sim-analyst` agent** — 专用仿真数据分析 subagent，隔离大量波形数据
- **`simulate` 超时** — 新增 `timeout` 参数，支持长时间 SPS 仿真
- **跨模型复制** — `block_add` 的 `source` 支持已加载模型路径
- **路径建议** — `source_not_found` 错误现在返回相似路径建议
- **`--json-file`** — 新 CLI 选项，从文件读取 JSON，解决 Windows 引号问题
- **UTF-8 输出** — 中文等非 ASCII 字符正确显示

### 兼容性

无破坏性变更。所有现有调用方式完全兼容。批量模式为纯增量功能。
