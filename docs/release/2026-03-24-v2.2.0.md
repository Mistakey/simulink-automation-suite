# Release v2.2.0

Release date: 2026-03-24

## Summary

`v2.2.0` publishes a `simulink-analyzer` read-analysis agent as a first-class plugin component. Read-only analysis workflows (topology scanning, block search, connection tracing, parameter inspection) now run in an isolated subagent context, keeping the main conversation free of intermediate CLI output. The existing `simulink-automation` skill is refocused on write operations, session management, and quick lookups, with an explicit Responsibility & Handoff contract governing routing between skill and agent.

This is a product architecture release, not a capability expansion — no new CLI actions, parameters, or error codes are introduced.

## Highlights

- New published agent `simulink-analyzer`: autonomously explores model topology, traces connections, audits parameters, and returns structured findings via a fixed six-section output envelope (Context, Answer, Evidence, Actions Performed, Limitations, Suggested Followup).
- Explicit Responsibility & Handoff contract in `SKILL.md` with three routing buckets: direct handling (session, schema, highlight, single-param inspect, writes), delegation to agent (scan, find, connections, full inspect, multi-step analysis), and composite request decomposition (analyze-then-write).
- Agent dispatch requires explicit resolved session and model context — the agent never self-discovers via session or list_opened actions.
- Agent is declared in `plugin.json` via explicit file path (`./agents/simulink-analyzer.md`), following the repository's manifest policy for agents.
- 18 new contract tests: agent definition contract (6), manifest agents contract (4), handoff contract (6), README agent coverage (2).
- READMEs (EN/ZH) updated with dual-capability narrative: read analysis via agent, write automation via skill.
- Schema version bumped to `2.2`; 12 registered actions (unchanged from v2.1.0 — no CLI contract expansion).
- 233 unit tests passing (18 new tests, net gain from 215 baseline).

## Compatibility / Upgrade Notes

- Plugin version is now `2.2.0`; schema version is `2.2`.
- No breaking changes to existing CLI actions — all v2.1.x payloads remain valid.
- Schema version `2.2` does not reflect a CLI action contract change. It is bumped to satisfy the existing release policy that requires `schema_version == f"{plugin_major}.{plugin_minor}"`. The action set, field definitions, and error codes are identical to v2.1.0.
- The `simulink-automation` skill's Workflow Strategy section has been trimmed: multi-step read analysis guidance is now the agent's responsibility. Users who rely on the skill for deep analysis workflows should expect the main agent to dispatch the analyzer agent instead.
- Roadmap version reallocation: v2.2.0 was originally planned for Block Placement (`block_add`). That capability is now scheduled for v2.3.0.

## 中文说明

### 摘要

`v2.2.0` 将 `simulink-analyzer` 只读分析 agent 作为一等插件组件发布。只读分析工作流（拓扑扫描、模块搜索、连接追踪、参数审计）现在运行在隔离的 subagent 上下文中，不再污染主对话上下文。`simulink-automation` 技能聚焦于写操作、会话管理和快速查询，并通过显式的 Responsibility & Handoff 合约控制技能与 agent 之间的路由。

这是一次产品架构发布，而非能力扩展——没有引入新的 CLI 动作、参数或错误码。

### 重点变更

- 新发布 `simulink-analyzer` agent：自主探索模型拓扑、追踪连接、审计参数，通过固定的六段式输出信封返回结构化结果。
- `SKILL.md` 新增 Responsibility & Handoff 合约，定义三种路由策略：直接处理、委派给 agent、复合请求分解。
- Agent dispatch 要求显式提供 session 和 model 上下文，agent 不会自行发现。
- 18 个新合约测试，233 个测试全部通过。
- Schema 版本升至 `2.2`，12 个注册动作（与 v2.1.0 一致，无 CLI 合约变更）。

### 兼容性说明

- 无破坏性变更，所有 v2.1.x 负载仍然有效。
- 路线图版本重新分配：v2.2.0 原计划用于 Block Placement（`block_add`），现推迟至 v2.3.0。
