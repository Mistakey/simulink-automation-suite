# v2.5.1 Release Notes

## Summary

Patch release addressing three backlog items: block positioning support, simulation parameter overrides, and structured mask parameter output.

## Highlights

- **F-004 `block_add` position & auto-layout**: `block_add` now accepts a `position` array (`[left, top, right, bottom]`) to place blocks at specific coordinates, plus an `auto_layout` flag to run `Simulink.BlockDiagram.arrangeSystem` after adding. Prevents blocks from stacking at default coordinates.
- **F-007a `simulate` temporary overrides**: `simulate` now accepts `stop_time` and `max_step` parameters that override solver settings for that run only, without modifying the model. Useful for diagnostic short simulations.
- **B-002 `inspect` mask value structuring**: For masked blocks, `inspect` now parses `MaskValueString` and pairs it with `MaskNames` into a structured `mask_values` dictionary. Eliminates the need to manually cross-reference pipe-separated value strings.

## Compatibility / Upgrade Notes

- All changes are additive. No breaking changes to existing action schemas or response shapes.
- New optional fields: `position`, `auto_layout` on `block_add`; `stop_time`, `max_step` on `simulate`.
- New optional response field: `mask_values` on `inspect` (present only for masked blocks), `overrides` on `simulate` (present only when overrides are used), `position` on `block_add` (present only when position is specified).

## Validation

- 351 unit tests passing
- Schema introspection verified: new fields registered correctly
- No docs contract violations

## 中文说明

### 摘要

补丁版本，解决三个 backlog 条目：模块定位支持、仿真参数临时覆盖、掩码参数结构化输出。

### 亮点

- **F-004 `block_add` 支持 position**：新增 `position` 数组参数指定模块坐标，`auto_layout` 标志自动排布。避免所有模块堆叠在默认位置。
- **F-007a `simulate` 临时覆盖**：新增 `stop_time` 和 `max_step` 参数，仅对当次仿真生效，不修改模型本身。便于诊断性短仿真。
- **B-002 `inspect` 掩码值结构化**：对带掩码的模块，自动解析 `MaskValueString` 并与 `MaskNames` 配对输出为 `mask_values` 字典，无需手动拆分竖线分隔字符串。

### 兼容性

- 所有变更为新增，无破坏性改动。
