# v2.4.0 Complete Modeling Workflow Design

Status: Approved (2026-03-25)

## Goal

AI can connect block ports, verify model structure, and close models — completing the full create-to-close workflow: `model_new → block_add → line_add → set_param → model_update → model_save → model_close`.

This version adds three actions across two safety tiers:

| Action | Module | Safety Tier |
|--------|--------|-------------|
| `line_add` | `line_add.py` | Checked Mutation |
| `model_close` | `model_close.py` | Operational (with dirty guard) |
| `model_update` | `model_update.py` | Operational |

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Port identification | String mode only (`block/port`) | AI can infer port numbers from block-type knowledge; no extra queries needed; handle mode adds round-trips with no unique benefit |
| Field structure | Decomposed (`src_block` + `src_port`) | Type safety (port is integer); no format convention for AI to learn; independent field validation; agent-first philosophy |
| model_close dirty behavior | Refuse by default, `force: true` to override | Consistent with set_param's `dry_run=true` default; writes default conservative |
| model_update transport | `set_param(model, 'SimulationCommand', 'update')` | Standard MATLAB approach; exposed as independent action for agent clarity |
| Module organization | One action per module | Codebase convention: `model_new.py`, `model_open.py`, `model_save.py`, `block_cmd.py` each export one action. New files: `line_add.py`, `model_close.py`, `model_update.py` |
| line_add rollback | Deferred (`available: false`) | `line_delete` not available until v2.5.0; same pattern as block_add |
| close_system MATLAB call | Always pass `0` flag to suppress save dialog | Agent-first CLI must never trigger interactive prompts; dirty guard at CLI level replaces MATLAB dialog |
| Block name validation | `src_block`/`dst_block` must not contain `/` | These are local names within the model, not paths; `/` would create ambiguous port strings |

## Action 1: `line_add` — Checked Mutation

### Schema

```python
# simulink_cli/actions/line_add.py

DESCRIPTION = "Connect two block ports with a signal line."

FIELDS = {
    "model":     {"type": "string",  "required": True,  "default": None,
                  "description": "Target model or subsystem path."},
    "src_block": {"type": "string",  "required": True,  "default": None,
                  "description": "Source block name (local to model, must not contain '/')."},
    "src_port":  {"type": "integer", "required": True,  "default": None,
                  "description": "Source output port number."},
    "dst_block": {"type": "string",  "required": True,  "default": None,
                  "description": "Destination block name (local to model, must not contain '/')."},
    "dst_port":  {"type": "integer", "required": True,  "default": None,
                  "description": "Destination input port number."},
    "session":   {"type": "string",  "required": False, "default": None,
                  "description": "MATLAB session name override."},
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "block_not_found", "port_not_found",
    "line_already_exists", "verification_failed", "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)
   Failure → engine_unavailable / no_session / session_not_found / session_required

2. PRECONDITION: model is loaded
   get_param(model, "Handle")
   Failure → model_not_found

3. PRECONDITION: source block exists
   get_param("{model}/{src_block}", "Handle")
   Failure → block_not_found (details: {"block": src_block, "role": "source"})

4. PRECONDITION: destination block exists
   get_param("{model}/{dst_block}", "Handle")
   Failure → block_not_found (details: {"block": dst_block, "role": "destination"})

5. EXECUTE
   result = add_line(model, "{src_block}/{src_port}", "{dst_block}/{dst_port}")
   Extract line_handle = result["value"]
   MATLAB rejects invalid port   → port_not_found
   MATLAB rejects occupied input → line_already_exists
   Other exception               → runtime_error

6. VERIFY
   get_param(line_handle, "Handle")  — pass float handle as target
   Failure → verification_failed (write_state: "attempted")

7. RESPOND
   {action, model, line_handle, verified: true,
    rollback: {available: false, note: "line_delete not yet implemented (v2.5.0)",
               session: <if explicit>}}
```

Note: `session` is conditionally included in rollback only (matching block_add pattern), not in the top-level response.

### Success Response

```json
{
  "action": "line_add",
  "model": "mymodel",
  "line_handle": 145.0003,
  "verified": true,
  "rollback": {
    "available": false,
    "note": "line_delete not yet implemented (v2.5.0)"
  }
}
```

## Action 2: `model_close` — Operational (Dirty Guard)

### Schema

```python
# simulink_cli/actions/model_close.py

DESCRIPTION = "Close a loaded Simulink model."

FIELDS = {
    "model":   {"type": "string",  "required": True,  "default": None,
                "description": "Model name to close."},
    "force":   {"type": "boolean", "required": False, "default": False,
                "description": "Close even if model has unsaved changes."},
    "session": {"type": "string",  "required": False, "default": None,
                "description": "MATLAB session name override."},
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "model_dirty", "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)

2. CHECK: model is loaded
   get_param(model, "Handle")
   Failure → model_not_found

3. CHECK: dirty state (unless force=true)
   get_param(model, "Dirty") → "on"/"off"
   "on" and force=false → model_dirty
     suggested_fix: model_save or model_close with force=true

4. EXECUTE
   close_system(model, 0)  — 0 suppresses interactive save dialog
   Exception → runtime_error

5. RESPOND
   {action: "model_close", model, force: <bool>}
```

Note: response uses `force` (matching input field name) for consistency with `set_param`'s `dry_run` pattern.

### Error: model_dirty

```json
{
  "error": "model_dirty",
  "message": "Model 'mymodel' has unsaved changes",
  "details": {},
  "suggested_fix": "Save first with {\"action\":\"model_save\",\"model\":\"mymodel\"} or close with {\"action\":\"model_close\",\"model\":\"mymodel\",\"force\":true}"
}
```

## Action 3: `model_update` — Operational

### Schema

```python
# simulink_cli/actions/model_update.py

DESCRIPTION = "Compile/update a loaded Simulink model diagram."

FIELDS = {
    "model":   {"type": "string",  "required": True,  "default": None,
                "description": "Model name to update/compile."},
    "session": {"type": "string",  "required": False, "default": None,
                "description": "MATLAB session name override."},
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "update_failed", "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)

2. CHECK: model is loaded
   get_param(model, "Handle")
   Failure → model_not_found

3. EXECUTE
   result = update_diagram(model)
   Extract warnings from result["warnings"]
   Compilation error → update_failed (includes MATLAB diagnostic in message)
   Other exception   → runtime_error

4. RESPOND
   {action: "model_update", model, warnings: [...]}
```

## Transport Layer

Add to `matlab_transport.py`:

```python
def add_line(engine, system, src, dst):
    return call(engine, "add_line", system, src, dst)

def close_system(engine, model):
    return call_no_output(engine, "close_system", model, 0)

def update_diagram(engine, model):
    return call_no_output(engine, "set_param", model, "SimulationCommand", "update")
```

Note: `close_system` always passes `0` as second argument to suppress MATLAB's interactive save dialog.

## Fake Engine Extensions

### FakeLineEngine (new)

Extends FakeBlockEngine capabilities:
- Tracks lines as `{(model, src_block, src_port, dst_block, dst_port)}` set
- Assigns incrementing float handles (e.g., `145.0001`, `145.0002`)
- `add_line(system, src, dst)`: parse `block/port` strings, check model loaded, check blocks exist, check dst port not occupied, return fake handle
- `get_param(line_handle, "Handle")`: accepts float handle as target, validates handle exists for verification

### FakeModelEngine extensions (for close/update)

Modify existing `FakeModelEngine`:
- Add dirty state tracking: `dirty_models: set`
- `get_param(model, "Dirty")` → `"on"` if in dirty set, `"off"` otherwise
- Modify existing `close_system(model)` to accept optional second arg (save flag), remove from loaded models
- `set_param(model, "SimulationCommand", "update")` → succeed if model loaded

## Collateral Updates

### model_new.py rollback activation

`model_new.py` currently has `rollback: {available: false, note: "model_close not yet implemented"}`. Since v2.4.0 implements `model_close`, update to:

```python
    "force": True,
    "available": True,
}
```
