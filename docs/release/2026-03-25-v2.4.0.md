# Release v2.4.0

Release date: 2026-03-25

## Summary

`v2.4.0` completes the full Simulink modeling workflow. AI agents can now execute the entire create-to-close cycle: `model_new → block_add → line_add → set_param → model_update → model_save → model_close`. This is the final sub-phase of Phase 1 ("From Zero to a Saveable Model").

## Highlights

- New `line_add` action (Checked Mutation): connect block ports with signal lines. Decomposed field design (`src_block` + `src_port` integer) for type safety. Precondition checks (model loaded, source/destination blocks exist) → execute → read-back verify → deferred rollback payload.
- New `model_close` action (Operational): close loaded models with dirty-state guard. Default refuses to close unsaved models; `force: true` overrides. `close_system` always passes `0` to suppress MATLAB's interactive save dialog.
- New `model_update` action (Operational): compile/update model diagram via `set_param(model, 'SimulationCommand', 'update')`. Extracts warnings from compilation. Distinguishes compilation errors (`update_failed`) from unexpected failures (`runtime_error`).
- `model_new` rollback activated: now returns `available: true` with `model_close` as the rollback action (previously deferred).
- New error codes: `model_dirty`, `port_not_found`, `line_already_exists`, `update_failed`.
- New modules: `simulink_cli/actions/line_add.py`, `model_close.py`, `model_update.py` + 3 transport wrappers.
- `FakeLineEngine` and `FakeModelEngine` extensions for behavior testing. `FakeWorkflowEngine` for end-to-end workflow testing.
- 55 new tests (line_add: 33, model_close: 14, model_update: 7, e2e: 1). 307 total tests passing.
- SKILL.md: 3 new actions in Direct Handling table and Workflow Strategy; new error codes in Recovery Routing.
- reference.md: response shapes for `line_add`, `model_close`, `model_update`.
- Schema version `2.4`, 16 registered actions (13 → 16).

## Compatibility / Upgrade Notes

- Plugin version `2.4.0`; schema version `2.4`.
- No breaking changes — all v2.3.x payloads remain valid.
- Three new actions are purely additive. Existing actions unchanged.
- `line_add` uses string-mode port identification only (`src_block`/`src_port` as separate fields). Port numbers are 1-based integers.
- `line_add` rollback is deferred (`available: false`) until `line_delete` ships in v2.5.0.
- `model_new` rollback is now active (`available: true`) — agents can auto-close newly created models on failure.

## 中文说明

### 摘要

`v2.4.0` 完成了完整的 Simulink 建模工作流。AI agent 现在可以执行从创建到关闭的完整流程：`model_new → block_add → line_add → set_param → model_update → model_save → model_close`。这是 Phase 1（"从零到可保存模型"）的最终子阶段。

### 重点变更

- 新增 `line_add` 动作（Checked Mutation）：连接模块端口。分解字段设计（`src_block` 字符串 + `src_port` 整数）确保类型安全。
- 新增 `model_close` 动作（Operational）：关闭模型，含脏状态守卫。默认拒绝关闭未保存模型，`force: true` 可强制关闭。
- 新增 `model_update` 动作（Operational）：编译/更新模型图，提取编译警告。
- `model_new` 回滚已激活：现在返回 `available: true`，使用 `model_close` 作为回滚动作。
- 新增错误码：`model_dirty`、`port_not_found`、`line_already_exists`、`update_failed`。
- 307 个测试全部通过（新增 55 个）。Schema 版本 `2.4`，16 个注册动作。

### 兼容性

- 无破坏性变更。所有 v2.3.x 请求负载仍然有效。
- 三个新动作为纯新增功能。
- `line_add` 回滚为延迟状态，待 v2.5.0 实现 `line_delete` 后激活。
