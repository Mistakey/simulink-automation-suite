# Release v2.1.0

Release date: 2026-03-21

## Summary

`v2.1.0` adds model lifecycle management to `simulink-edit`: AI agents can now create, open, and save Simulink models through three new actions (`model_new`, `model_open`, `model_save`). This is the first sub-phase of the v2.1.x roadmap, enabling full create-read-write-save workflows without leaving the CLI.

## Highlights

- New action `model_new`: create a Simulink model with precondition check (rejects `model_already_loaded`) and post-creation verification.
- New action `model_open`: open a model from file path or MATLAB path; idempotent — re-opening an already-open model is not an error.
- New action `model_save`: save a loaded model to disk with existence check before write.
- Three new transport wrappers (`new_system`, `open_system`, `save_system`) in `matlab_transport.py`.
- `model_new` returns a rollback payload with `available: false` (model_close not yet implemented).
- Schema version bumped to `2.1`; 11 registered actions (up from 8).
- New error codes: `model_already_loaded`, `model_save_failed`.
- Skill docs (`SKILL.md`, `reference.md`, `test-scenarios.md`), READMEs (EN/ZH), and docs contract tests all updated.
- 337 unit tests passing (18 new tests for model lifecycle actions).

## Compatibility / Upgrade Notes

- Plugin version is now `2.1.0`; schema version is `2.1`.
- No breaking changes to existing actions — all v2.0.x payloads remain valid.
- `model_new` rollback has `available: false` because `model_close` is not yet implemented. Manual MATLAB `close_system` is required to undo.
- `model_open` / `model_save` do not use dry-run semantics (unlike `set_param`). `model_open` is inherently idempotent; `model_save` is a straightforward persist operation.

## 中文说明

### 摘要

`v2.1.0` 为 `simulink-edit` 技能新增模型生命周期管理能力：AI 代理现在可以通过 `model_new`、`model_open`、`model_save` 三个新动作创建、打开和保存 Simulink 模型，实现完整的 创建-读取-修改-保存 工作流。

### 重点变更

- 新动作 `model_new`：创建模型，带前置检查（已加载则返回 `model_already_loaded`）和创建后验证。
- 新动作 `model_open`：从文件路径打开模型；幂等操作，重复打开不报错。
- 新动作 `model_save`：保存已加载的模型到磁盘。
- Schema 版本升至 `2.1`，注册动作从 8 个增加到 11 个。
- 新增错误码：`model_already_loaded`、`model_save_failed`。
- 337 个单元测试全部通过（新增 18 个模型生命周期测试）。

### 升级提示

- 与 v2.0.x 完全兼容，已有 payload 无需修改。
- `model_new` 的 rollback 标记为 `available: false`，因为 `model_close` 尚未实现，需手动在 MATLAB 中执行 `close_system` 撤销。
