# v2.5.0 Complete Editing + Simulation Design

Status: Approved (2026-03-26)

## Goal

AI can remove blocks/connections, run simulations, and deferred rollbacks activate — completing the full 14-action capability baseline.

This version adds three actions across two safety tiers:

| Action | Module | Safety Tier |
|--------|--------|-------------|
| `line_delete` | `line_delete.py` | Checked Mutation |
| `block_delete` | `block_delete.py` | Checked Mutation |
| `simulate` | `simulate_cmd.py` | Operational |

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| block_delete safety tier | Checked Mutation (not Full Guarded) | Full Guarded requires atomic rollback; block_delete destroys block + params + lines — multi-step restore is a recovery plan, not a rollback. Agent has inspect + connections to query before deleting. |
| block_delete rollback | `available: false` | Cannot atomically restore block with original params and connections. Add later if real usage demands it. |
| block_delete dry_run | Not implemented | Would duplicate inspect + connections capability. dry_run semantics ("old → new" diff) don't apply to deletion. |
| block_delete connected lines | Silent deletion, documented in SKILL.md | MATLAB `delete_block` natively removes connected lines. Reporting them would require pre-delete queries (essentially dry_run). SKILL.md guides agent to check connections before deleting. |
| block_delete field name | `destination` (matches block_add) | block_add rollback payload maps directly to block_delete schema with zero adaptation. |
| line_delete identification | Port pairs (matches line_add) | Symmetric with line_add; MATLAB `delete_line` accepts `(system, 'block/port', 'block/port')`; agent knows what it connected. |
| line_delete verification | No read-back (no-exception = success) | MATLAB `delete_line` either succeeds or throws; no silent failure mode. Unlike add operations which may silently fail. |
| simulate fields | `model` + `session` only | StopTime is a model-level param (set once via set_param, persisted in .slx). No convenience fields — keeps simulate pure. |
| simulate response | `{action, model, warnings}` only | SimulationOutput object is too large for agent context. Agent uses other tools for result inspection if needed (post-v2.5.0). |
| simulate blocking | Synchronous (inherent to MATLAB sim()) | Agent expects to wait for simulation. Timeout is a transport-level concern, not simulate-specific. |
| simulate suggested_fix | Not included in simulation_failed | Simulation failures have diverse causes; generic suggestions are not actionable. MATLAB diagnostic in `cause` field is sufficient. |
| simulate warnings | Transport-level only (lastwarn) in v2.5.0 | MATLAB `sim()` returns a SimulationOutput object containing embedded diagnostics (algebraic loop warnings, saturation warnings, etc.). Extracting these requires SimulationOutput API calls whose Python Engine serialization behavior is unverified. v2.5.0 surfaces only transport-level warnings from `_drain_warnings` (lastwarn mechanism). SimulationOutput warning extraction is deferred to post-v2.5.0 after live MATLAB verification. |
| simulate transport | `call(engine, "sim", model)` with nargout=1 | `sim()` has a return value (SimulationOutput); using nargout=0 may cause MATLAB errors. Action layer discards `result["value"]`, keeping only `result["warnings"]`. |
| Module naming | `simulate_cmd.py` | Avoids collision with Python builtins/keywords; consistent with `session_cmd.py` pattern. |
| Implementation order | line_delete → block_delete → simulate → rollback activation | Dependency chain: block_delete's fake engine benefits from delete_line semantics; rollback activation requires both delete actions to exist. |

## Action 1: `line_delete` — Checked Mutation

### Schema

```python
# simulink_cli/actions/line_delete.py

DESCRIPTION = "Delete a signal line between two block ports."

FIELDS = {
    "model":     {"type": "string",  "required": True,  "default": None,
                  "description": "Target model or subsystem path."},
    "src_block": {"type": "string",  "required": True,  "default": None,
                  "description": "Source block name (local to model, must not contain '/')."},
    "src_port":  {"type": "integer", "required": True,  "default": None,
                  "description": "Source output port number."},
    "dst_block": {"type": "string",  "required": True,  "default": None,
                  "description": "Destination block name (local to model, must not contain '/')."},
    "dst_port":  {"type": "integer", "required": True,  "default": None,
                  "description": "Destination input port number."},
    "session":   {"type": "string",  "required": False, "default": None,
                  "description": "MATLAB session name override."},
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "block_not_found", "line_not_found",
    "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)
   Failure → engine_unavailable / no_session / session_not_found / session_required

2. PRECONDITION: model is loaded
   get_param(model, "Handle")
   Failure → model_not_found
     suggested_fix: "Open the model first: {\"action\":\"model_open\",\"path\":\"<model>.slx\"}"

3. PRECONDITION: source block exists
   get_param("{model}/{src_block}", "Handle")
   Failure → block_not_found (details: {"block": src_block, "role": "source"})
     suggested_fix: "Check spelling. Use find to list blocks in the model."

4. PRECONDITION: destination block exists
   get_param("{model}/{dst_block}", "Handle")
   Failure → block_not_found (details: {"block": dst_block, "role": "destination"})
     suggested_fix: "Check spelling. Use find to list blocks in the model."

5. EXECUTE
   delete_line(model, "{src_block}/{src_port}", "{dst_block}/{dst_port}")
   Line not found → line_not_found
     suggested_fix: "Use connections to verify the line exists before deleting."
   Other exception → runtime_error

6. RESPOND (no read-back verification — no-exception = success)
   {action, model, src_block, src_port, dst_block, dst_port,
    rollback: {available: true, action: "line_add", ...port fields...,
               session: <if explicit>}}
```

### Success Response

```json
{
  "action": "line_delete",
  "model": "mymodel",
  "src_block": "Sine",
  "src_port": 1,
  "dst_block": "Gain",
  "dst_port": 1,
  "rollback": {
    "action": "line_add",
    "model": "mymodel",
    "src_block": "Sine",
    "src_port": 1,
    "dst_block": "Gain",
    "dst_port": 1,
    "available": true
  }
}
```

### Error: line_not_found

```json
{
  "error": "line_not_found",
  "message": "No line found from 'Sine/1' to 'Gain/1' in 'mymodel'.",
  "details": {"src": "Sine/1", "dst": "Gain/1", "model": "mymodel"},
  "suggested_fix": "Use connections to verify the line exists before deleting."
}
```

## Action 2: `block_delete` — Checked Mutation

### Schema

```python
# simulink_cli/actions/block_delete.py

DESCRIPTION = "Delete a block from a loaded Simulink model."

FIELDS = {
    "destination": {
        "type": "string",  "required": True,  "default": None,
        "description": "Full block path in model (e.g. 'my_model/Gain1').",
    },
    "session": {
        "type": "string",  "required": False, "default": None,
        "description": "MATLAB session name override.",
    },
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "block_not_found",
    "verification_failed", "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)

2. PRECONDITION: model is loaded
   model_root = destination.split("/")[0]
   get_param(model_root, "Handle")
   Failure → model_not_found
     suggested_fix: "Open the model first: {\"action\":\"model_open\",\"path\":\"<model>.slx\"}"

3. PRECONDITION: block exists
   get_param(destination, "Handle")
   Failure → block_not_found
     suggested_fix: "Check spelling. Use find to list blocks in the model."

4. EXECUTE
   delete_block(destination)
   Exception → runtime_error

5. VERIFY — reverse check (block should NOT exist)
   get_param(destination, "Handle")
   If succeeds → verification_failed (block still exists)
     suggested_fix: "Block may still exist. Use inspect to verify state."
   If throws → success (block is gone)

6. RESPOND
   {action, destination, verified: true,
    rollback: {available: false, note: "...", session: <if explicit>}}
```

### Success Response

```json
{
  "action": "block_delete",
  "destination": "demo/Gain1",
  "verified": true,
  "rollback": {
    "available": false,
    "note": "Block deletion also removes connected lines. Use block_add to re-create with library defaults."
  }
}
```

## Action 3: `simulate` — Operational

### Schema

```python
# simulink_cli/actions/simulate_cmd.py

DESCRIPTION = "Run simulation on a loaded Simulink model."

FIELDS = {
    "model": {
        "type": "string",  "required": True,  "default": None,
        "description": "Model name to simulate.",
    },
    "session": {
        "type": "string",  "required": False, "default": None,
        "description": "MATLAB session name override.",
    },
}

ERRORS = [
    "engine_unavailable", "no_session", "session_not_found", "session_required",
    "model_not_found", "simulation_failed", "runtime_error",
]
```

### Execute Flow

```
1. CONNECT — safe_connect_to_session(session)

2. CHECK: model is loaded
   get_param(model, "Handle")
   Failure → model_not_found

3. EXECUTE
   result = sim(model)
   Extract warnings from result["warnings"]
   Simulation error → simulation_failed (includes MATLAB diagnostic in details.cause)
   Other exception → runtime_error

4. RESPOND
   {action, model, warnings}
```

### Success Response

```json
{
  "action": "simulate",
  "model": "demo",
  "warnings": []
}
```

### Error: simulation_failed

```json
{
  "error": "simulation_failed",
  "message": "Simulation of 'demo' failed.",
  "details": {"model": "demo", "cause": "...MATLAB diagnostic..."}
}
```

## Transport Layer

Add to `matlab_transport.py`:

```python
def delete_line(engine, system, src, dst):
    """Delete a signal line between two block ports."""
    return call_no_output(engine, "delete_line", system, src, dst)

def delete_block(engine, block_path):
    """Delete a block from a loaded model."""
    return call_no_output(engine, "delete_block", block_path)

def sim(engine, model):
    """Run simulation on a loaded model."""
    return call(engine, "sim", model)
```

## Fake Engine Extensions

### FakeLineEngine (extend)

Add `delete_line(system, src, dst)`:
- Parse `block/port` strings
- Check model loaded, line exists in `_lines` by matching `(system, src, dst)`
- Remove from `_lines` and `_dst_ports`
- Raise if not found

### FakeBlockEngine (extend)

Add `delete_block(block_path)`:
- Check block exists in `_blocks`
- Remove from `_blocks`
- Raise if not found

### FakeModelEngine (extend)

Add `sim(model)`:
- Check model loaded
- Return model name (simulating successful simulation)
- Raise if model not loaded

### FakeWorkflowEngine (extend)

Add `delete_line`, `delete_block`, `sim` — composing the above behaviors for e2e tests.
