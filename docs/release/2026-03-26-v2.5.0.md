# v2.5.0 — Complete Editing + Simulation

`v2.5.0` completes the 14-action capability baseline. AI agents can now remove blocks and signal lines, run simulations, and use activated rollbacks for structural edits. This is the final planned feature release — future work is driven by real usage.

## Highlights

- **`line_delete`** (Checked Mutation) — disconnect two block ports by specifying the same port-pair used in `line_add`. Rollback provides a ready-to-submit `line_add` payload.
- **`block_delete`** (Checked Mutation) — remove a block from a loaded model with reverse verification. Note: MATLAB natively removes connected signal lines when a block is deleted. Rollback is `available: false` (atomic restore not possible).
- **`simulate`** (Operational) — run `sim(model)` and return warnings. SimulationOutput is intentionally discarded to preserve agent context window. Only transport-level warnings (lastwarn) are surfaced in v2.5.0.
- **Rollback activation** — `line_add` and `block_add` rollbacks now point to `line_delete` and `block_delete` respectively, with `available: true`.
- **Roadmap retired** — `docs/roadmap.md` deleted. Safety tier definitions merged into `simulink_cli/.claude/CLAUDE.md`. README roadmap section simplified.
- **Test cleanup** — 29 self-referential tests removed (fake engine unit tests + function-existence checks). 331 tests remain.

## New Actions

| Action | Safety Tier | Fields |
|--------|-------------|--------|
| `line_delete` | Checked Mutation | model, src_block, src_port, dst_block, dst_port, session |
| `block_delete` | Checked Mutation | destination, session |
| `simulate` | Operational | model, session |

## New Error Codes

| Code | Action | Meaning |
|------|--------|---------|
| `line_not_found` | `line_delete` | No line exists between the specified ports |
| `simulation_failed` | `simulate` | MATLAB simulation error (diagnostic in `details.cause`) |

## Compatibility / Upgrade Notes

- **`line_add` rollback shape changed**: the rollback payload no longer contains `line_handle` or `note`. It now uses port-pair fields (`src_block`, `src_port`, `dst_block`, `dst_port`) matching `line_delete` schema, with `available: true`.
- **`block_add` rollback shape changed**: the rollback payload no longer contains `note`. It now has `available: true` pointing to `block_delete`.
- Agents relying on `rollback.available == false` or `rollback.note` in `line_add`/`block_add` responses should update their logic.
- Schema version bumped from `2.4` to `2.5`. All existing actions unchanged.

## 中文说明

v2.5.0 完成了 14 项基础能力。AI agent 现在可以删除模块和信号线、运行仿真，并使用已激活的回滚功能。

### 新增功能

- `line_delete` — 断开两个端口之间的信号线（Checked Mutation）
- `block_delete` — 从模型中删除模块，MATLAB 会同时删除连接的信号线（Checked Mutation）
- `simulate` — 运行仿真，返回警告信息（Operational）
- `line_add` 和 `block_add` 的回滚功能已激活（`available: true`）

### 兼容性说明

- `line_add` 的 rollback 结构变更：不再包含 `line_handle` 和 `note`，改用端口对字段
- `block_add` 的 rollback 结构变更：不再包含 `note`，`available` 改为 `true`
- Schema 版本从 2.4 升级到 2.5，现有 action 无变化
