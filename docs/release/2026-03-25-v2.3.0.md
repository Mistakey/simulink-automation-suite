# Release v2.3.0

Release date: 2026-03-25

## Summary

`v2.3.0` adds `block_add`, the first structural editing action. AI agents can now place library blocks into loaded Simulink models, completing the third sub-phase of the Phase 1 roadmap ("From Zero to a Saveable Model"). Combined with existing `model_new` and `model_save`, agents can create models that contain actual blocks — not just empty shells.

## Highlights

- New `block_add` action (Checked Mutation safety tier): precondition checks (model loaded, source exists, destination vacant) → execute → read-back verify → deferred rollback payload.
- Source validation via `get_param(source, "Handle")` precondition — structural check, no MATLAB error-text parsing. Robust across MATLAB versions.
- Deferred rollback: `block_delete` not yet implemented (Phase 3). Rollback payload includes manual MATLAB undo command.
- New error codes: `source_not_found`, `block_already_exists`.
- New module: `simulink_cli/actions/block_cmd.py` + transport wrapper `add_block()`.
- `FakeBlockEngine` in test fakes for behavior testing without MATLAB.
- 18 new tests (behavior: 10 execute + 3 validate + 4 fake engine + 1 transport). 252 total tests passing.
- SKILL.md: `block_add` added to Direct Handling table and Workflow Strategy; new error codes in Recovery Routing.
- reference.md: `block_add` response shapes documented.
- Schema version `2.3`, 13 registered actions (12 → 13).

## Compatibility / Upgrade Notes

- Plugin version `2.3.0`; schema version `2.3`.
- No breaking changes — all v2.2.x payloads remain valid.
- New action `block_add` is purely additive. Existing actions unchanged.
- Position parameter intentionally omitted: MATLAB auto-places blocks. Use `set_param` to adjust position afterward; use `Simulink.BlockDiagram.arrangeSystem` for layout.
- `block_add` rollback is deferred (`available: false`) until `block_delete` ships in Phase 3.

## 中文说明

### 摘要

`v2.3.0` 新增 `block_add` 动作——首个结构编辑能力。AI agent 现在可以向已加载的 Simulink 模型中添加库模块，完成 Phase 1 路线图的第三个子阶段。结合已有的 `model_new` 和 `model_save`，agent 能创建包含实际模块的模型，而非仅仅空壳。

### 重点变更

- 新增 `block_add` 动作（Checked Mutation 安全层级）：前置条件检查（模型已加载、源存在、目标未占用）→ 执行 → 读回验证 → 延迟回滚负载。
- Source 验证通过 `get_param(source, "Handle")` 前置检查实现——结构化检查，不解析 MATLAB 错误文本，跨版本稳健。
- 新增错误码：`source_not_found`、`block_already_exists`。
- 252 个测试全部通过（新增 18 个）。
- Schema 版本 `2.3`，13 个注册动作。

### 兼容性

- 无破坏性变更。所有 v2.2.x 请求负载仍然有效。
- `block_add` 为纯新增功能。位置参数暂未提供，MATLAB 自动放置。
- 回滚为延迟状态（`available: false`），待 Phase 3 实现 `block_delete` 后激活。
